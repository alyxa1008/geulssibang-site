# 구구단 암송 검사 설계안 (2026-07-10)

> 목표: 받아쓰기처럼 "시험 보는 동안 사이트가 선생님 역할"을 하는 두 번째 체류형 도구.
> 공식: **부모의 반복 노동(문제 내주기·채점)을 사이트가 대체**하되, 결과가 인쇄 학습지(본업)로 이어진다.

## 1. 포지셔닝

- **URL**: `/gugudan/` (최상위 — badaseugi·plan과 동급의 독립 도구)
- **검색 의도 분담**: `/math/gugudan/`(기존 랜딩)은 "구구단 문제지·학습지"(인쇄),
  `/gugudan/`은 "구구단 외우기·시험·테스트"(화면). 서로 잠식 없음, 상호 링크.
- **title**: `구구단 외우기 시험 — 문제를 불러주는 구구단 테스트 무료 | 글씨방`
- **정체성 가드**: 게임이 아니라 **검사 도구**. 점수·오답 진단·학습지 연결. 캐릭터/이펙트 최소.
- **톤**: 응원 톤 유지 — "6단만 몇 번 더 하면 완성이에요" (불안 조장 금지, audit 10번 준수)

## 2. 교육과정 근거 (2022 개정, 2학년 2학기 '곱셈구구')

- 검정교과서 공통 학습 순서: **2·5단 → 3·6단 → 4·8단 → 7·9단 → 1단과 0의 곱**
  (묶어 세기 쉬운 단부터). 이 순서를 단 선택 프리셋으로 제공 = 경쟁 게임 사이트에 없는 차별점.
- 1단·0의 곱은 옵션 (기본 꺼짐 — 암송 수요의 중심은 2~9단).

## 3. UX 흐름 (받아쓰기 패턴 복제)

### 설정 화면 (panel)
| 항목 | 옵션 | 기본 |
|---|---|---|
| 단 선택 | 2~9 다중 토글 + 프리셋 칩(①2·5단 ②3·6단 ③4·8단 ④7·9단 / 전체) | 전체 |
| 문제 수 | 10 / 20 / 선택한 단 전부 | 10 |
| 순서 | 섞어서 / 차례대로(2×1부터) | 섞어서 |
| 답하는 방법 | **말로 답하기**(정답 공개 후 ⭕❌ 터치 채점) / **숫자로 입력**(키패드, 자동 채점) | 말로 답하기 |
| 생각할 시간 | 3초 / 5초 / 8초 (말하기 모드의 정답 공개 대기) | 5초 |
| 문제 읽어주기 | 켬 / 끔(화면만 — 조용한 환경) | 켬 |

### 시험 화면
- 진행바 + "3 / 10" (받아쓰기 examProg/examBar 패턴)
- 큰 문제 카드: `7 × 6 = ?` + TTS "7 곱하기 6은?" (숫자를 그대로 넘기면 한국어 보이스가 자연 낭독 — 별도 수사 변환 불필요)
- **말하기 모드**: 카운트다운 → 정답 `42` 공개 + 낭독 → ⭕/❌ 큰 버튼 (부모나 아이가 터치)
- **입력 모드**: 화면 숫자 키패드(0-9, 지움, 확인) → 자동 채점, 틀리면 정답 잠깐 표시 후 다음
- 중단 버튼 상시 노출. 이탈·중단 시 `clearTimeout` + `speechSynthesis.cancel` (받아쓰기와 동일한 정리 규약)

### 결과 화면 (본업 연결 지점)
- 점수 N / M + 단별 정오 요약 ("6단 2문제, 7단 1문제를 다시 볼까요?")
- 틀린 문제 목록 (7×6=42 형태로 정답 노출 — 복기용)
- CTA 3개:
  1. **틀린 문제만 다시 시험** (오답 재시험 — 체류 연장의 핵심)
  2. **틀린 단 학습지 인쇄** → 기존 math 딥링크 v2 `[2,"gugudan",level,0,dan,...]` 재사용, 단별 버튼
  3. 처음부터 다시

## 4. 기술 설계

- **파일**: `/gugudan/index.html` 단일 파일(인라인 JS, ~500줄) — 사이트 패턴 유지(한 페이지 한 책임)
- **재사용**: common.js의 `koVoice/speakKo·koVoiceMissing`(음성), `track`(GA), `b64e/d`(공유), `showToast`
- **타이머 규약**: 모든 setTimeout은 `exam.timer` 한 곳에 저장, 상태 전환·중단 시 반드시 clear (받아쓰기 검증된 패턴)
- **공유 링크**: `#s=` b64 `[1, 단문자열("23456789"), 문제수, 순서, 답방식, 생각시간, 소리]` — v1 버전 넘버 포함
- **wireDeepLinks 확장**: `.gobtn[data-quiz-dan]` 유형 1개 추가 (랜딩·결과 CTA에서 단 지정 링크 생성, 인라인 스크립트 금지 원칙 유지)
- **GA 이벤트**: `exam_start {tool:"gugudan", dans, n}` (기존 이벤트명 재사용), 완료 시 `quiz_done {score, total}`
- **인쇄 없음**: 이 도구 자체는 화면 전용. 인쇄는 결과 CTA로 math 도구에 위임 (인쇄 엔진 중복 제작 금지)

## 5. SEO·플러밍

- 페이지 콘텐츠: 사용법 3단계 / 교과서 순서로 외우는 법(2·5→3·6→4·8→7·9 근거 설명) /
  "외우기 전에 개념부터" → `/math/gugudan/` 개념 섹션 링크 / FAQ / JSON-LD(WebApplication+FAQPage)

- 홈: tools 카드 7번째 추가(⏱ 구구단 시험) + guides-box 수학 줄
